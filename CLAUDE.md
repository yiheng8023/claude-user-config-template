# Claude Code Global Guidance Template

This is a public-safe starter `CLAUDE.md` for a private Claude Code user
configuration repository.

Replace this file with your own rules in your private repository.

Recommended baseline:

- Do not expose secrets, credentials, local paths, account state, or private
  memory in public output.
- Prefer small, verifiable changes.
- Explain uncertainty and verify before claiming completion.
- Keep real project facts anchored in the current repository, not in old memory.
- Treat hooks as optional automation aids, not as authority sources.

Private notes, personal preferences, and project-specific memory belong in your
private repository only.
