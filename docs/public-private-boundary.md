# Public / Private Boundary

## Public-safe

- Generic directory layout.
- Placeholder configuration.
- Example scripts that do not require private credentials.
- Documentation and validation rules.

## Private-only

- Real memory.
- Personal prompts, preferences, and project notes.
- Local machine paths.
- OAuth, credentials, tokens, cookies, private keys, and passwords.
- Account-specific MCP configuration.
- Runtime logs, caches, session data, telemetry, and history.

The template may be public. Real configuration should stay private.
